# Figma 디자인 구현

## 역할

제품 디자인 시스템과 GUI 감사 결과를 Figma의 편집 가능한 산출물로 옮긴다.

- 제품 정체성·IA·토큰·제품 컴포넌트 읽기
- 감사 결과와 개선 범위를 Figma 작업 단위로 변환
- 기존 라이브러리와 컴포넌트를 찾아 재사용
- 네이티브 프레임, 오토 레이아웃, 변수, 스타일, 컴포넌트와 변형으로 구현
- 구현 결과를 다시 확인하고 다음 작업을 위해 memory를 갱신

이 스킬은 제품 정체성을 결정하거나 코드 GUI를 직접 개선하는 스킬이 아니다. 시스템의 방향을 바꾸려면 `$design-product`, 실제 프로젝트 GUI의 세부 조정은 `$design-audit`로 넘긴다.

## 입력과 선행 조건

1. 프로젝트 루트의 `design-product/manifest.json`을 확인한다.
2. `design-product`가 없으면 먼저 `$design-product`로 제품 시스템을 설정한다.
3. `design-product`의 정체성, 토큰, 레이아웃 규칙, 제품 컴포넌트와 현재 상태를 읽는다.
4. 대상 화면에 대한 `design-audit` 결과가 있으면 함께 읽는다. 감사 결과가 없으면 필요한 범위의 감사부터 `$design-audit`으로 수행한다.
5. 제품 시스템과 감사 결과가 충돌하면 임의로 덮어쓰지 말고 어느 기준을 채택할지 확인한다.

Figma 도구의 사전 조건과 구체적인 작업 방식은 `references/figma.md`를 따른다. 도구 호출 전 연결된 Figma MCP 또는 제품 환경이 요구하는 사용 규칙을 확인한다.

## 표준 흐름

1. **컨텍스트를 압축해 읽는다**
   - 필요한 경우 `render_context.py`를 실행하고 대상 화면과 관련된 진실의 원천만 추가로 읽는다.
2. **구현 범위를 정한다**
   - 새 화면, 기존 화면 수정, 제품 컴포넌트, 토큰·변수 매핑 중 무엇을 만들지 정한다.
   - 화면 유형, 사용자 목표, 주요 행동, 콘텐츠 계층과 비목표를 짧게 계획한다.
3. **Figma 구조를 확인한다**
   - 기존 파일·페이지·라이브러리·컴포넌트·변수를 검색한다.
   - 프로젝트에 합의된 페이지 구조가 있으면 그대로 사용하고, 없으면 필요한 최소 구조만 제안한다.
4. **네이티브 구조로 구현한다**
   - 제품 컴포넌트를 도메인 역할로 이름 붙인다.
   - 토큰을 변수와 스타일에 매핑하고, 반응형 동작을 오토 레이아웃과 제약 조건으로 표현한다.
   - 화면을 납작한 스크린샷으로 붙이지 않는다.
5. **감사 결과를 반영한다**
   - `design-audit`의 발견 사항이 있으면 구현 과정에서 해결 여부를 확인한다.
   - 정체성, IA, 제품 특수성, 토큰 준수, 편집성, 공유성을 점검한다.
6. **인계 상태를 기록한다**
   - 구현한 페이지·프레임·컴포넌트와 미완료 항목을 기록한다.
   - 장기적으로 재사용할 결정만 `design-product/memory/`에 추가한다.

## 하드 룰

- `design-product`의 정체성·토큰·컴포넌트 규칙을 Figma에서 임의로 재정의하지 않는다.
- `design-audit`의 세부 개선 결과를 반영하되, 범위를 넘어선 전면 재설계는 `$design-product`로 넘긴다.
- 가능한 경우 기존 라이브러리와 컴포넌트를 재사용하고 중복 원시 컴포넌트를 만들지 않는다.
- 프레임, 오토 레이아웃, 변수, 스타일, 컴포넌트와 변형을 사용해 계속 편집 가능한 구조를 만든다.
- 레이어 이름은 도메인 역할을 나타내야 하며, `Frame 123` 같은 기본 이름을 남기지 않는다.
- 확인하지 못한 Figma 파일, 페이지, 연결 상태 또는 완료 여부를 사실처럼 보고하지 않는다.

## Figma 도구가 없을 때

파일이 갱신되었다고 가장하지 않는다. 대신 다음을 작성한다.

- IA·컴포넌트 계획
- 페이지·프레임·레이어 구조
- 토큰과 Figma 변수 매핑
- 감사 결과와 해결 상태
- Figma MCP가 연결되었을 때 실행할 정확한 다음 작업

## 완료 결과

- 참조한 `design-product`와 `design-audit` 산출물
- 생성·수정한 Figma 페이지·프레임·컴포넌트
- 토큰·변수·라이브러리 매핑
- 편집성·반응형·주요 흐름 검증 결과
- memory에 기록한 결정과 남은 작업

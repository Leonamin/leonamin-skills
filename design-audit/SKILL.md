# GUI 디자인 감사

## 역할

이미 운영 중인 프로젝트의 GUI를 좁은 범위에서 검토하고 개선한다.

- 특정 화면이나 사용자 흐름의 사용성·시각적 문제 찾기
- 기존 제품 정체성, 토큰과 컴포넌트 규칙의 위반 찾기
- 카드, 여백, 계층, 문구, 상태, 반응형 동작의 세부 조정
- 발견 사항을 우선순위화하고 최소 변경으로 개선
- 변경 결과와 남은 예외를 프로젝트 memory에 기록

이 스킬은 전체 디자인 시스템을 새로 만들거나 제품의 방향을 다시 정하는 스킬이 아니다. 문제가 시스템의 정체성·토큰·컴포넌트 경계 자체에 있으면 `$design-product`로 넘긴다.

## 다른 디자인 스킬과의 경계

- 초기 설정 또는 전면 재설계: `$design-product`
- 기존 GUI의 세부 감사와 개선: `$design-audit`
- 디자인 결과물을 Figma의 편집 가능한 화면·컴포넌트로 구현: `$figma-design`

`design-audit`은 `design-product`의 결과물을 읽고, 필요하면 실제 GUI 코드를 수정한다. Figma에 직접 쓰지는 않으며, Figma 작업은 감사 결과를 `$figma-design`에 인계한다.

## 시작 전 확인

스킬 디렉터리 기준으로 스크립트 경로를 해석한다.

1. 프로젝트 루트에서 `design-product/manifest.json`을 확인한다.
2. 파일이 없으면 먼저 `$design-product`로 초기 설정하도록 안내하고, 근거 없는 세부 감사를 시작하지 않는다.
3. 다음 명령으로 결정론적 감사를 실행한다.

~~~bash
python3 <skill-dir>/scripts/audit_design.py --path <repo>
~~~

CI처럼 오류 발견 시 실패해야 하는 경우에만 `--strict`를 추가한다. 결과는 신호로 보고, 제품 맥락과 실제 화면을 함께 확인한다.

관련 컨텍스트가 필요하면 다음을 먼저 읽는다.

~~~bash
python3 <design-product-skill-dir>/scripts/render_context.py --path <repo>
~~~

그 다음 `design-product/manifest.json`이 가리키는 정체성, 토큰, 레이아웃, 컴포넌트 규칙과 현재 상태 중 감사 범위에 해당하는 파일만 읽는다. 감사 기준과 보고 형식은 `references/audit.md`를 따른다.

## 표준 흐름

1. **범위를 고정한다**
   - 대상 화면·컴포넌트·플로우와 이번 작업에서 바꾸지 않을 영역을 정한다.
   - 초기화, 브랜드 방향 변경, 토큰 체계 교체가 필요하면 `$design-product`로 전환한다.
2. **발견 사항을 수집한다**
   - 스크립트 결과와 실제 코드·화면·스크린샷·브라우저 상태를 대조한다.
   - 구조적 문제, 시각적 문제, 콘텐츠 문제, 접근성·반응형 문제를 구분한다.
3. **우선순위를 정한다**
   - 사용자 목표를 막는 문제를 높게, 일관성이나 세련됨만 개선하는 문제를 낮게 둔다.
   - 각 발견 사항에 파일·행 또는 화면 위치, 근거, 영향과 권장 조치를 적는다.
4. **작은 변경을 제안한다**
   - 기존 토큰과 제품 컴포넌트를 재사용한다.
   - 선택지가 있으면 2~3개를 비교하고, 영향 범위가 가장 작은 해결책을 권장한다.
   - 사용자가 보고만 요청했으면 구현하지 않는다. 개선을 요청했으면 승인된 범위만 구현한다.
5. **결과를 검증한다**
   - 같은 감사 명령을 다시 실행한다.
   - 가능하면 실제 화면의 주요 흐름, 키보드 접근성, 좁은 화면과 넓은 화면을 확인한다.
   - 해결하지 않은 발견 사항은 이유와 후속 작업으로 남긴다.
6. **기억을 갱신한다**
   - 오래 유지될 예외·패턴·결정만 `design-product/memory/`에 기록한다.
   - 감사 요약은 프로젝트의 `design-product/audits/last-audit.md`에 반영한다.

## 하드 룰

- 작은 범위의 개선을 위해 제품 정체성, 토큰 체계 또는 전체 IA를 임의로 갈아엎지 않는다.
- 진실의 원천에 정의된 토큰과 제품 컴포넌트를 우선한다.
- 원시 hex, 장식용 그라디언트, 과도한 카드·그림자·radius는 제품 맥락 없이 추가하지 않는다.
- 접근성, 반응형 동작, 로딩·오류·빈 상태를 시각적 취향보다 먼저 확인한다.
- 가짜 데이터, 후기, 수치, 신뢰 표식을 만들어 문제를 가리지 않는다.
- 감사 결과를 개선했다고 주장하려면 실제 파일 변경과 검증 증거가 있어야 한다.
- Figma를 수정하거나 Figma 결과물을 감사하는 작업은 `$figma-design`의 도구·흐름으로 넘긴다.

## 완료 결과

보고 또는 구현을 마칠 때 다음을 남긴다.

- 감사 범위
- 우선순위별 발견 사항과 근거
- 구현한 변경 파일 또는 권장 변경안
- 재실행한 감사·화면 검증 결과
- memory와 `last-audit.md`에 기록한 내용
- `$design-product` 또는 `$figma-design`으로 넘길 후속 작업

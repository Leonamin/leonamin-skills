# 산출물

감사 가능성, 인계 또는 리뷰가 필요한 작업에는 가벼운 산출물을 남긴다. 저장소에 이미 규칙이 있으면 그 규칙을 우선한다.

## 계약

고위험이거나 티켓에 묶인 작업은 구현 전에 계약을 만들거나 기존 계약을 확인한다. 저장소에 규칙이 없으면 다음과 같은 경로를 사용할 수 있다.

~~~text
.agents/tasks/{task-id}/contract.md
~~~

다음 내용을 포함한다.

- 목표
- 비목표
- 허용 파일 또는 모듈
- 금지 파일 또는 모듈
- 제약과 가정
- 검증 계획
- 필요한 증거
- 완료 조건
- 필요한 경우 승인 기준

## 증거

구현이 복잡하거나 리뷰를 거쳤거나 위임된 경우 증거를 남긴다.

~~~text
.agents/tasks/{task-id}/evidence.md
~~~

다음 내용을 포함한다.

- 변경 요약
- 변경 파일
- 실제 실행한 검증 명령
- 명령 결과와 실패 내용
- UI 작업의 스크린샷 또는 녹화 경로
- 마이그레이션·배포·운영 영향
- 남은 위험

## 리뷰

읽기 전용 리뷰 결과는 저장소 규칙이 없을 때 다음 경로를 사용할 수 있다.

~~~text
.agents/tasks/{task-id}/review.md
~~~

다음 내용을 포함한다.

- 심각도 순으로 정렬한 발견 사항
- 열린 질문
- 검증 공백
- 범위 또는 계약 위반
- 완료 판단: Complete, Needs fixes 또는 Blocked

## 인계

세션, 에이전트 또는 워크트리를 넘나드는 작업에는 인계 메모를 사용한다.

- 사용자 요청
- 저장소, 브랜치와 워크트리
- diff 기준 또는 최신 커밋
- 범위와 비목표
- 완료한 작업
- 남은 작업
- 실행한 명령과 결과
- 실행 중인 로컬 서비스와 중지 방법
- 알려진 위험 또는 차단 요소

## 완료로 보고하지 말아야 하는 경우

- 필요한 승인이 없다.
- 필요한 검증이 실행되지 않았다.
- 증거가 필요한 작업인데 증거가 없다.
- 리뷰가 필요한 작업인데 리뷰가 없다.
- 시각적 증거가 필요한 UI 작업인데 스크린샷이 없다.
- 차단 수준의 리뷰 발견 사항이 해결되지 않았다.
- 운영·배포 영향이 있는 변경을 필요한 환경에서 검증하지 않았다.
